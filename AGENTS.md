# 🤖 Agent Guide to Rotkeeper

Welcome to the Rotkeeper repository. If you are an autonomous AI agent, LLM, or programmatic assistant exploring this project, this file serves as your "nickel tour" of the architecture and workflow.

## 🧭 The Core Philosophy
Rotkeeper is a terminal-based toolkit for rendering, archiving, and preserving the remains of Markdown-driven projects.
- **Inputs** are Markdown files (called "tombs").
- **Outputs** are rendered HTML files and `.tar.gz` archives.
- **Rituals** are bash scripts that perform operations (rendering, packaging, scanning).

## 📂 Architecture
- `rotkeeper.sh`: The single entry point and CLI router. Always start here. Run `./rotkeeper.sh help` for commands.
- `home/`: The user's workspace.
  - `home/content/`: Where all Markdown source files live. You should write `.md` files here.
  - `home/assets/`: Static assets (CSS, images) that will be bundled.
- `output/`: The destination for rendered HTML.
- `bones/`: The internal system directory.
  - `bones/scripts/`: The actual bash scripts that execute the rituals (e.g., `rc-render.sh`).
  - `bones/config/`: System configurations (like `render-flags.yaml`).
  - `bones/archives/`: Where `.tar.gz` versions and releases are stored.

## 🚀 Quickstart for Agents
If you are tasked with creating a page or testing the framework:
1. **Initialize the workspace** by running: `./rotkeeper.sh init`
2. **Create a Markdown file** in `home/content/` (e.g., `home/content/test.md`).
3. **Add YAML frontmatter** at the very top (must start with `---`). Example:
   ```yaml
   ---
   title: "Test Page"
   slug: test
   template: rotkeeper-blog.html
   ---
   ```
3. **Write Markdown content** below the frontmatter.
4. **Render** the site by running: `./rotkeeper.sh render`
5. **Verify** your output in the `output/` directory (e.g., `output/test.html`).

## Decentralized Ingestion
If you are generating content or auditing the system, you can use Rotkeeper's ingestion pipeline to permanently save your work:
1. Ensure your generated markdown files have proper YAML frontmatter. Include a `report.md` or `audit.md` summarizing your methods and findings.
2. Run `./rotkeeper.sh pack --content`. This bundles `home/content/` into a `.tar.gz` (excluding redundant docs).
3. The resulting `.tar.gz` archive is safely preserved in `bones/archive/` and can be ingested by the central repository via `./rotkeeper.sh ingest`.

## ⚠️ Known Friction Points
- Do not edit `bones/config/render-flags.yaml` unless you are absolutely sure. It is often autogenerated.
- Ensure your YAML frontmatter *always* starts with `---`. If omitted, Pandoc will treat your metadata as regular paragraph text in the HTML output.

Good luck!
